Building H-lang: Project Setup

Part 1: Initialize the Go Project

Overview

In this tutorial, we'll set up the H-lang compiler project structure. The compiler is written in Go and follows a standard Go project layout.

1Create the project directory

mkdir -p h-lang
cd h-lang

2Initialize Go module

go mod init github.com/yourusername/h-lang

This creates go.mod:

module github.com/yourusername/h-lang

go 1.23

3Create the directory structure

mkdir -p cmd/hlc
mkdir -p pkg/lexer
mkdir -p pkg/parser
mkdir -p pkg/ast
mkdir -p pkg/types
mkdir -p pkg/codegen
mkdir -p examples

Project Structure

h-lang/
├── cmd/
│   └── hlc/              # Compiler CLI
│       └── main.go
├── pkg/
│   ├── lexer/            # Tokenizer
│   │   ├── token.go      # Token types
│   │   └── lexer.go      # Lexer implementation
│   ├── parser/           # AST builder
│   │   └── parser.go
│   ├── ast/              # AST types
│   │   └── ast.go
│   ├── types/            # Type checker (future)
│   │   └── checker.go
│   └── codegen/          # C code generator
│       └── codegen.go
├── examples/
│   ├── hello.hl
│   └── demo.hl
├── go.mod
└── README.md

Package Responsibilities

Package Purpose
cmd/hlc Compiler CLI entry point
pkg/lexer Converts source text into tokens
pkg/ast Defines AST node types
pkg/parser Builds AST from tokens
pkg/types Type checking (future)
pkg/codegen Generates C code from AST

4Create a placeholder main.go

// cmd/hlc/main.go
package main

import "fmt"

func main() {
    fmt.Println("H-lang compiler v0.1.0")
}

5Verify the setup

go build ./cmd/hlc
./hlc

You should see: H-lang compiler v0.1.0

Compilation Pipeline

The H-lang compiler follows a standard pipeline:

Source (.hl)
    │
    ▼
┌─────────┐  characters   ┌─────────┐  tokens    ┌────────┐
│  Read   │──────────────▶│  Lexer  │───────────▶│ Parser │
│  File   │               │         │            │        │
└─────────┘               └─────────┘            └────────┘
                                                      │
                                                      │ AST
                                                      ▼
┌─────────┐  binary       ┌─────────┐  C code   ┌─────────┐
│  gcc/   │◀──────────────│ Codegen │◀──────────│  Type   │
│  clang  │               │         │           │  Check  │
└─────────┘               └─────────┘           └─────────┘
Why transpile to C?
  • Leverage decades of gcc/clang optimisations
  • Instant portability to all platforms
  • Manual memory management maps directly to C
  • Focus on language design, not code generation
